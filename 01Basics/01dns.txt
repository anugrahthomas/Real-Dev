DNS TTL (Time To Live): 
â³ DNS TTL (Time To Live)
DNS TTL is the amount of time a DNS record is allowed to stay in cache before it must be refreshed.
It is set in seconds.
Example: TTL = 3600 â†’ record is cached for 1 hour.
- After TTL expires, the DNS server must ask again for updated information.
Why TTL matters
- High TTL (e.g., 24 hours)
    - Faster performance
    - Fewer DNS lookups
- Slower updates if changes are made
    - Low TTL (e.g., 5 minutes)
    - Changes update faster
    - More DNS traffic
ðŸ‘‰ Think of TTL like an expiration date on stored information.
