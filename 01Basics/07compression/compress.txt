Http Compression: 
HTTP compression is a way to reduce the size of data sent over the network from server to client. Smaller responses mean:
- Faster page loads
- Less bandwidth usage
- Better performance on slow networks

⚠️Important: Compression works on text-based resources, like:
- HTML, CSS, JavaScript
- JSON, XML, SVG
- Sometimes plain text files
It doesn’t help much with images, video, or already compressed files (like .jpg, .png, .mp4).

🧩 How HTTP Compression Works
1. Browser advertises support in request header:
- Accept-Encoding: gzip, deflate, br
• gzip → common compression format
• deflate → legacy compression
br → Brotli (modern, highly efficient)
2. Server compresses the response using one of the requested methods.
3. Server sends compressed content with header:
- Content-Encoding: gzip
4. Browser decompresses automatically and renders it.
Flow Intuition:
- Browser: “I can handle gzip or Brotli.”
- Server: “OK, I’ll send gzip to save bandwidth.”
- Browser unzips it and uses it—transparent to the user.

🧩 Common Compression Methods
| Method        | Notes                                                  |
| ------------- | ------------------------------------------------------ |
| `gzip`        | Widely supported, good compression for most text       |
| `deflate`     | Older, less common, smaller compression improvement    |
| `br` (Brotli) | Best compression for web, supported by modern browsers |
| `compress`    | Obsolete, don’t use                                    |
👉 Use Brotli for HTTPS sites (modern browsers support it), fallback to gzip

☀️ Compression + Caching:
- Compressed resources can be cached.
- Browser or CDN stores the compressed version → reused for future requests.

🧩 Compresion can be done using:
1. Server configuration (e.g., Apache, Nginx)
2. Application-level libraries (e.g., Node.js middleware)
3. CDNs (Content Delivery Networks) often handle compression automatically.

eg:
const express = require('express');
const app = express();

{ const compression = require('compression');
  app.use(compression()); // compress all responses }

app.get('/', (req, res) => {
  res.send('Hello, compressed world!');
});
app.listen(3000);



ZLIB:
- zlib is a compression library that implements the DEFLATE algorithm.
- DEFLATE → the compression algorithm
- zlib → a library that implements DEFLATE + adds a small wrapper format
- gzip → DEFLATE + different wrapper format

The Core Algorithm: DEFLATE
DEFLATE combines two techniques:
1. LZ77 (pattern matching): finds and replaces repeated sequences of data with references to a single copy of that data.
2. Huffman Coding (bit-level compression): assigns shorter codes to more frequent symbols and longer codes to less frequent symbols, reducing the overall size of the data.
👉 LZ77 finds patterns, Huffman coding compresses the result further

zlib wrapper: [Header][Compressed Data][Checksum]

| Format      | Has Header | Has Checksum | Used In      |
| ----------- | ---------- | ------------ | ------------ |
| raw deflate | ❌          | ❌            | internal use |
| zlib        | ✅ small    | ✅ Adler-32   | PNG          |
| gzip        | ✅ larger   | ✅ CRC32      | HTTP         |

👉 zlib works in streams, written in C


In production do not compress in server because it blocks event loop
1. reverse proxy (e.g., Nginx) can handle compression efficiently
2. CDNs often handle compression automatically, reducing load on origin server


Real Production Setup (Senior-Level)

Modern architecture:

↓

↓
WAF
↓
Load Balancer
↓
Nginx
↓
Node (business logic only)
↓
Redis
↓
Database

Node should NOT:
- Compress
- Handle TLS
- Serve static files
- Handle rate limiting ideally

It should focus on:
- Business logic
- Data processing
- API responses


Real Production Consequence
Under load:
- P95 latency increases: it is the latency at which 95% of requests are faster, but the slowest 5% can be much slower due to compression overhead
- P99 latency explodes: the slowest 1% of requests can take significantly longer, leading to timeouts and poor user experience
- Health checks may fail
- Auto-scaling may trigger
- Cost increases
